# Инструкции для Codex

Эти правила применяются ко всем чатам и задачам внутри этого репозитория.

- Для поиска и анализа кода сначала использовать граф `codebase-memory-mcp`: `search_graph`, `trace_path`, `get_code_snippet`, затем при необходимости `query_graph` и `get_architecture`; к обычному поиску переходить для строк, конфигурации и когда данных графа недостаточно.
- Вести журнал изменений в `CHANGELOG.md`.
- После каждого значимого изменения добавлять краткую запись в журнал: дата, что изменено и зачем.
- После важных изменений делать `git commit` и `git push`.
- Перед коммитом проверять `git status`, чтобы не включить посторонние пользовательские изменения.
- Если push невозможен из-за состояния удаленного репозитория или прав доступа, явно сообщить об этом пользователю.
- Папка `kornix base/` содержит копии внешних репозиториев `meteo` и `kornix_site`: не изменять ее содержимое, использовать только для чтения и не добавлять в git.
